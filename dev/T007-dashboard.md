# T007 — Dashboard: Визуализация портфеля и опционов

## Описание
Создать интерактивный дашборд с 4 вкладками для визуализации позиций по активам, опционам, рекомендаций модели и общей сводки портфеля. Обновление данных каждые 30-60 сек.

## Подзадачи

### T007.1 — Структура проекта и бэкенд
- [ ] Создать директорию `dashboard/` в проекте
- [ ] Написать FastAPI-сервер: чтение CSV, агрегация данных, отдача JSON
- [ ] Точки: `/positions`, `/options`, `/recommendations`, `/summary`
- [ ] Настроить автозапуск (systemd/supervisor или просто `uvicorn` в фоне)

### T007.2 — Базовый фронтенд: навигация + Лист 1 (Активы)
- [ ] HTML-скелет с табами, Chart.js
- [ ] Лист 1: таблица позиций по активам
- [ ] Графики ценовых уровней с разметкой PnL, просадка
- [ ] Автообновление через fetch

### T007.3 — Лист 2: Опционные позиции
- [ ] Таблица: символ, страйк, DTE, цена, PnL, Greeks (текущие vs при входе)
- [ ] Визуализация тета-распада (график decay по дням)
- [ ] Cost of carry за день

### T007.4 — Лист 3: Рекомендации модели
- [ ] Отображение рекомендаций по страйкам
- [ ] Моделирование PnL/delta/gamma/theta для каждого варианта
- [ ] Сравнение слоев (ближний vs дальний)

### T007.5 — Лист 4: Сводка портфеля
- [ ] PnL по активам, по опционам, общий
- [ ] Net Greeks: delta, gamma, theta, vega
- [ ] Прогноз на день: theta decay, potential PnL range
- [ ] Визуальная сводка (графики + ключевые цифры)

### T007.6 — Полировка и деплой
- [ ] Responsive дизайн
- [ ] Цветовая маркировка: зелёный = прибыль, красный = убыток
- [ ] Обработка ошибок API, fallback при недоступности
- [ ] Интеграция в запуск проекта (один скрипт стартует бэк + фронт)
