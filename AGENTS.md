# 📋 Правила работы над HedgeModel

> **Обязательны к чтению при каждом обращении.**

---

## 1. Задачи фиксируются немедленно

Когда выявляем задачу, которую нужно решить:
- **Сразу** записываем в `900-open-tasks.md`
- **Никогда** не оставляем "в голове"
- Если задача найдена — показываем пользователю: "Задача X найдена, записал в 900-open-tasks"

## 2. Структурируем, не создаём хаос

- Пользователь думает, я структурирую
- Не предлагаю новые структуры без объяснения, зачем это нужно
- Не удаляю то, что пользователь определил, пока не спрошу
- Не создаю TODO там, где пользователь определил ответ

## 3. Идеи и мозговой штурм

- Любая идея, возникшая в разговоре → **сразу записываю**
- Не жду, пока пользователь попросит
- Создаю раздел "Идеи" в файле проекта
- Не забываю записывать — это ценность

## 4. Не забываем о договорённостях

- При каждом обращении читаю `AGENTS.md`
- Проверяю, что выполняю правила
- Если вижу, что задача из `900-open-tasks.md` ещё не решена — напоминаю

## 5. Конкретика вместо абстракций

- Не предлагаю "настроить что-то" без конкретики
- Не создаю файлы-заглушки без содержания
- Всё, что делаю — конкретный файл с конкретным содержанием

## 6. Анализ перед предложениями

- **НИКОГДА** не предлагаю изменения в файле, пока не прочитал его полностью
- Перед любым предложением: читаю файл → анализирую структуру → проверяю, что уже есть
- Если предлагаю что-то, что уже есть — это ошибка, которую я признаю и исправляю
- Не предлагаю новые колонки, скрипты, файлы без проверки: а точно ли этого нет?
- Не предлагаю добавить то, что пользователь уже определил

**Алгоритм:**
1. Прочитать файл
2. Понять текущую структуру
3. Только потом предложить что-то
