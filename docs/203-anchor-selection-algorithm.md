# 203 — Алгоритм выбора опционов для дальнего слоя (Anchor Layer)

> Алгоритм подбора двух страйков (Core + Tail) для защиты от коррекций 15–30%. Цель — найти опционы с **максимальной выпуклостью (гаммой) на каждый вложенный доллар**.

---

## Роль Anchor Layer

Обеспечивает **асимметричную защиту** при экстремальных движениях рынка. Опционы стоят недорого, но при срабатывании дают непропорционально большую прибыль.

**Диапазон страхования:**
- **Нижняя граница:** S = Spot × 0.70 (падение -30%, глубокий краш)
- **Верхняя граница:** S = Spot × 0.85 (падение -15%, начальная коррекция)

**Почему 15–30%?**
- 15% — первый серьёзный откат, когда Active Layer уже сработал
- 30% — глубокая коррекция, защита критична
- Anchor Layer — защита от медвежьей коррекции

---

## Шаг 1: Расчёт бюджета Anchor Layer

См. [[200-budget-allocation]]

```
PositionSize = SOL_qty × Avg_Buy_Price
TotalBudget  = PositionSize × GlobalBudgetPct
AnchorBudget = TotalBudget × AnchorLayerPct
```

**Почему Spot:** все метрики считаются от текущей цены — так мы оцениваем актуальную защиту.

---

## Шаг 2: Фильтрация кандидатов

Отбираем PUT-опционы по критериям из [[201-selection-criteria#anchor]] + антипаттерны:

**Базовый диапазон:**
- Delta: 0.05 – 0.20 (глубоко OTM)
- DTE: >= 12 дней (долгосрочная защита)
- Distance: <= 35% (не берём "супер-дешёвые на всякий случай")

**Антипаттерны (НЕ покупаем):**
- ❌ Delta < 0.05 — слишком дёшево, нет эффекта при падении
- ❌ Bid/Ask спред > 10% — плохая ликвидность
- ❌ IV > 150% — завышенная волатильность, переплата

**Gamma-фильтр:**
- Строим кривую Gamma(S) для каждого кандидата
- Находим S_at_max (где Gamma максимальна)
- **Отбрасываем**, если S_at_max далеко от диапазона [0.70×Spot, 0.85×Spot]
- Критерий: `|S_at_max - Spot| / Spot > 0.30` или `|S_at_max - Spot| / Spot < 0.15`

**Формула Gamma:**
```
Gamma_put(S) = φ(d1) / (S × σ × √T)
где φ — функция плотности нормального распределения
```

**Результат:** список кандидатов, где пик гаммы попадает в диапазон страхования.

## Шаг 3: Рейтинг эффективности капитала (CE-Ranking)

**Цель:** не рейтинг опционов, а рейтинг **эффективности использования капитала**. Какой опцион даёт больше выпуклости (гаммы) на каждый вложенный доллар?

### 3.1 Capital Efficiency (CE)

**Главный критерий сортировки.**

```
CE = GammaProtection / Premium

GammaProtection = Σ Gamma(S) × ΔS
где S ∈ [Spot × 0.70, Spot × 0.85]
      ΔS = шаг $1 (для точности можно использовать $0.5)
```

**Смысл:** сколько суммарной защитной силы (гаммы) мы получаем на каждый вложенный доллар.

**Чем выше CE — тем эффективнее опцион.**

### 3.2 Метрики покрытия

- **Baseline Strength** = CE при S = Spot × 0.80 (коррекция -20%)
- **Tail Strength** = CE при S = Spot × 0.70 (краш -30%)
- **GammaProtection** = суммарная гамма по всему диапазону [0.70×Spot, 0.85×Spot]

### 3.3 Пример расчёта CE

Пусть:
- Spot = $75
- Premium = $1.50
- K = $65, DTE = 30, IV = 75%

**GammaProtection** (от $52.50 до $63.75):

| S | Gamma(S) | ΔS | Gamma × ΔS |
|---|----------|----|------------|
| 63.75 | 0.003 | $1 | 0.003 |
| 62.75 | 0.004 | $1 | 0.004 |
| 61.75 | 0.005 | $1 | 0.005 |
| ... | ... | ... | ... |
| 52.50 | 0.008 | $1 | 0.008 |

**Σ GammaProtection = 0.120**

```
CE = 0.120 / 1.50 = 0.080
```

**Вывод:** каждый вложенный доллар даёт 0.080 суммарной гаммы.

### 3.4 CE-таблица кандидатов

| Опцион | K | Premium | GammaProtection | CE | Ранг |
|--------|---|---------|----------------|----|------|
| A | $65 | $2.10 | 0.160 | **0.076** | 3 |
| B | $60 | $1.50 | 0.120 | **0.080** | 2 |
| C | $55 | $0.90 | 0.090 | **0.100** | 1 |

## Шаг 4: Выбор Core (K1) и Tail (K2) с бюджетным ограничением

### 4.1 Разделение диапазона на зоны

```
Зона Core: [Spot × 0.80, Spot × 0.85]  (коррекция -15% до -20%)
Зона Tail: [Spot × 0.70, Spot × 0.80]  (краш -20% до -30%)
```

### 4.2 Выбор Core — K1

Core — опцион с **максимальным CE** в зоне Core (ближняя граница диапазона).

```
K1 = argmax_K [CE при S ∈ [0.80×Spot, 0.85×Spot]]
```

**Характеристики Core:**
- Высокая CE при начальной коррекции (-15% до -20%)
- Отвечает за первый удар, срабатывает сразу
- Обычно более высокий страйк (ближе к текущей цене)

### 4.3 Выбор Tail — K2

Tail — опцион с **максимальным CE** в зоне Tail (дальняя граница диапазона).

```
K2 = argmax_K [CE при S ∈ [0.70×Spot, 0.80×Spot]]
```

**Характеристики Tail:**
- Максимальная CE при глубоком падении (-20% до -30%)
- Срабатывает при экстремальном падении
- Обычно более низкий страйк (дальше от текущей цены)

### 4.4 Бюджетное ограничение

**Важно:** не более **40% бюджета** на один страйк.

```
CoreBudget = AnchorBudget × 0.40
TailBudget = AnchorBudget - CoreBudget
```

Если CE(Core) > CE(Tail):
- Core получает максимум 40% бюджета
- Tail получает остаток (но тоже не более 40%, остальное — резерв)

### 4.5 Пример комбинации:

| Опцион | Зона | CE | Бюджет | Расход |
|--------|------|----|--------|--------|
| B ($60) | Core | 0.080 | 40% | $2.10 |
| C ($55) | Tail | 0.100 | 60% | $3.15 |

**Итого:** Core 40% бюджета (CE=0.080) + Tail 60% бюджета (CE=0.100)

## Шаг 5: Проверка PnL-профиля

После выбора комбинации (K1, K2) проверяем итоговый PnL-профиль:

```
PnL(S) = n1 × [BS_Price(S, K1, T1, IV1) - Premium1] + n2 × [BS_Price(S, K2, T2, IV2) - Premium2]
```

**Критерии:**
- ✅ При S = Spot × 0.85 → PnL ≥ 0 (коррекция -15% покрывается)
- ✅ При S = Spot × 0.80 → PnL > 0 (коррекция -20% выгодна)
- ✅ При S = Spot × 0.70 → PnL > 0 (краш -30% выгоден)
- ✅ PnL растёт при движении вниз (выпуклость подтверждена)

**Корректировка:**
- Если PnL < 0 при -20% → увеличить n2 (Tail)
- Если PnL < 0 при -15% → увеличить n1 (Core)
- Если PnL не растёт → выбрать другой K1/K2

---

## Шаг 6: Расчёт количества контрактов

### Бюджет
```
TotalCost = n1 × Premium(K1) + n2 × Premium(K2) ≤ AnchorBudget
```

### Delta-хедж
При падении на целевую просадку (S = Spot × 0.80):

```
Delta_hedge = n1 × |Delta(K1)| + n2 × |Delta(K2)|
DeltaRatio = Delta_hedge / SOL_qty
```

**Целевая дельта:** >= 80% при целевой просадке.

### Распределение бюджета
- **Core (K1):** не более 40% бюджета
- **Tail (K2):** остаток (до 60%)

### Оптимизация
Ищем (n1, n2), которые:
1. Укладываются в AnchorBudget
2. DeltaRatio >= 80% при падении на 20%
3. Core не превышает 40% бюджета
4. PnL-профиль удовлетворяет критериям (Шаг 5)

---

## Шаг 7: Условия закрытия Anchor Layer

См. [[101-anchor-layer]] и [[104-active-closing-rules]]

**Правила закрытия:**
- ❌ Delta > 0.85 → закрыть (глубоко ITM, защита уже сработала)
- ❌ Delta > 0.90 → закрыть (зона исчерпана)
- ❌ Delta < 0.10 → закрыть (глубоко OTM, гамма мертва)
- ❌ DTE < 7 → закрыть (theta-распад убивает стоимость)
- ✅ Рынок упал на 15–30% → удерживать (защита сработала, фиксируем прибыль)
- ✅ Смена тренда → Close (не ролл, так как Anchor — долгосрочный хедж)

**Главный принцип:** закрытие — это реакция на выход из рабочей зоны, а не фиксация прибыли/убытка.
