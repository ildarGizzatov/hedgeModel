# Выбор опционов для Anchor Layer (dev/T006-anchor-selection)

> Алгоритм подбора двух страйков (Core + Tail) для дальнего слоя защиты.

---

## Роль Anchor Layer

Обеспечивает **асимметричную защиту**: опционы стоят недорого, но при срабатывании дают непропорционально большую прибыль. Работает при экстремальных движениях рынка.

---

## Шаг 0: Расчёт бюджета Anchor

См. [[001-input-parameters]]

```
PositionSize = SOL_qty × Avg_Buy_Price
TotalBudget  = PositionSize × GlobalBudgetPct
AnchorBudget = TotalBudget × AnchorLayerPct
```

**Почему Avg_Buy_Price:** хедж покупается для защиты от просадки относительно цены входа.

---

## Шаг 1: Фильтрация кандидатов

Отбираем PUT-опционы по критериям из [[201-selection-criteria#anchor]] + антипаттерны:

**Дополнительное ограничение:** Distance <= 35% — не берём "супер-дешёвые на всякий случай".

**Антипаттерны (НЕ покупаем):**
- ❌ Delta < 0.05 — слишком дешёвые, нет эффекта
- ❌ Bid/Ask спред > 10% — плохая ликвидность

---

## Шаг 2: Матрица эффективности E(S, K)

Для каждого кандидата считаем эффективность по формуле:

```
E(S, K) = (BS_Price(S_target, K, T, IV) - Premium) / Premium
```

Где:
- **S_target** — целевая цена (уровень падения от цены покупки)
- **Premium** — текущая цена опциона
- **BS_Price** — Black-Scholes цена опциона при целевой цене

**Ключевые уровни:**

| Уровень | Описание | Формула |
|---------|----------|---------|
| Baseline (-20%) | Обычная коррекция, хедж включается | S = AvgBuy × 0.80 |
| Tail (-35%) | Экстремальный краш | S = AvgBuy × 0.65 |

**Метрики для каждого страйка:**

- **Baseline Strength** = E(-20%) — эффективность при нормальной коррекции
- **Acceleration** = E(-35%) - E(-20%) — как быстро E растёт при углублении кризиса
- **ΓE** = ускорение ускорения (второй порядок) — аналог gamma для эффективности

---

## Шаг 3: Выбор Core (K1) и Tail (K2)

### Core — K1 (Ядро)

```
K1 = argmax K (E(-20%))
```

Выбираем страйк с **максимальной эффективностью при падении на 20%**. Это «щит», который срабатывает сразу и отражает первый удар.

### Tail — K2 (Хвост)

```
K2 = argmax K (Acceleration) при K ≠ K1
```

Выбираем страйк с **максимальным ускорением** (рост E между -20% и -35%). Это «усиление», которое срабатывает при экстремальном падении.

**Характеристики:**
- K1: высокая E при умеренном падении, мёртв в спокойном рынке
- K2: слабый старт, но максимальный прирост при углублении кризиса

---

## Шаг 4: Расчёт количества контрактов

### Бюджет
```
TotalCost = n1 × Premium(K1) + n2 × Premium(K2) ≤ AnchorBudget
```

### Delta-хедж
При падении на целевую просадку (S = AvgBuy × 0.80):

```
Delta_hedge = n1 × |Delta(K1)| + n2 × |Delta(K2)|
DeltaRatio = Delta_hedge / SOL_qty
```

**Целевая дельта:** >= 80% при целевой просадке.

### Распределение бюджета
- **Core (K1):** максимум 60% бюджета
- **Tail (K2):** остальное

### Оптимизация
Ищем (n1, n2), которые:
1. Укладываются в AnchorBudget
2. DeltaRatio >= 80% при целевой просадке
3. Core составляет максимум 60% бюджета
4. Minimize отклонение DeltaRatio от 100%
