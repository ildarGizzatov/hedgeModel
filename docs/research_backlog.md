*Этот файл предназначен для фиксации всех идей, гипотез и вопросов, возникших в процессе исследования, которые еще не получили окончательного решения или реализации.*

## 📂 Непроработанные задачи (Pending Implementation)

### 1. Механизм распределения бюджета слоев (Layer Budget Allocation)
* **Суть:** Как именно глобальный лимит (5–10%) дробится на конкретные суммы для Anchor, Adaptation и Active слоев? Должно ли это быть фиксированным проценту или динамическим в зависимости от волатильности?
* **Статус:** `Exploration`
* **Связь:** `docs/strategy_definition.md` (Глобальный Уровень управления капиталом).

### 2. Реализация «Загрузчика Профилей Активов» (Asset Profile Loader)
* **Суть:** Разработка программного модуля, который при старте системы подгружает границы зон (Игнорирование/Внимание/Хедж) для конкретного тикера (например, SOL).
* **Статус:** `Analysis`
* **Связь:** `docs/strategy_definition.md` (Динамика актива).

### 3. Определение ограничений для Среднего слоя (Adaptation Layer Constraints)
* **Суть:** Разработка критериев отсева (Delta, DTE, Max Drawdown Reduction), аналогичных тем, что мы применили для Дальнего слоя, с учетом его роли в стабилизации.
* **Статус:** `Analysis`
* **Связь:** `docs/strategy_definition.md`

### 4. Определение ограничений для Ближнего слоя (Active Layer Constraints)
* **Суть:** Разработка критериев максимизации Гаммы ($\Gamma$) и скорости PnL, чтобы слой работал как «двигатель прибыли», не превышая выделенный бюджет слоя.
* **Статус:** `Analysis`
* **Связь:** `docs/strategy_definition.md`

## 💡 Гипотезы для проверки (Hypotheses to Test)

### 1. Влияние динамического порога на эффективность (Dynamic Threshold vs Efficiency)
* **Гипотеза:** Изменение границ «зоны внимания» в зависимости от ATR (Average True Range) позволит снизить частоту ложных срабатываний хеджа и уменьшить общую стоимость стратегии.
* **Статус:** `Exploration`
* **Связь:** `docs/strategy_definition.md` (Динамика актива).
