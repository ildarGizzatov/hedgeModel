## Подбор опционов — правило

- Жёсткие критерии — **не менять**. Это основа модели.
- Показывать: оригинальные критерии → сколько попало → граничные опционы
- Смягчённый вариант — только для **анализа и поиска аналогов**, не для изменения модели
- Записывать **закономерности** — почему оригинальные не работают, что нужно скорректировать
- Пользователь принимает решение с контекстом

## Правило смягчения (принято 2026-06-01 15:40)

- Смягчение ≠ отклонение на 50% — грубая ошибка была с DTE 20-30 вместо 25-45
- **Допустимое отклонение: не более 10%**
- Смягчённый вариант: только **предлагать**, не принимать за пользователя
- **Всегда ждать согласия** пользователя на любое смягчение
- Без согласия — критерии остаются жёсткими

## Диагностика Anchor (спот $80.97, 2026-06-01)

- Оригинал (DTE 25-45): **0 шт.**
- Граничный: DTE=24 (на 1 день меньше, ~4% от границы 25 — допустимо ≤10%)
  - SOL-26JUN26-70-P ($0.71), SOL-26JUN26-72-P ($0.99)
- Паттерн: макс DTE по цепочке = 24
- **Требует решения пользователя:** принять DTE=24 или оставить 0 шт.

## Диагностика Active (спот $80.74, 2026-06-02)

- Оригинал (Delta 0.38-0.55, DTE 5-10, Distance 0-5%): **1 шт.** — SOL-12JUN26-80-P
- Граничный (в пределах 10%): DTE=11 (если есть опционы с DTE=11 вблизи границы 10)
- Паттерн: только один Put с Delta 0.38-0.55 на нужном DTE
