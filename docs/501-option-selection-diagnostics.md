# 🔍 Правила подбора опционов — диагностика

> Правила фильтрации и валидации кандидатов в опционы.

---

## Жёсткие критерии

Жёсткие критерии отбора из [[201-selection-criteria]] — **не менять**. Это основа модели.

## Показывать всегда

- Оригинальные критерии
- Сколько кандидатов попало
- Граничные опционы (отклонение ≤10%)

## Смягчение

- Допустимое отклонение: **не более 10%**
- Смягчённый вариант — только для **предложения**, не принятия за пользователя
- **Всегда ждать согласия** пользователя
- Без согласия — критерии остаются жёсткими

## Обязательное правило

При обновлении доски опционов **всегда** обновлять `current_price` в таблице `positions` (БД).
Происходит автоматически после каждого запуска `pipeline.py fetch`.
