# 210 — Алгоритм выбора опционов ближнего слоя (Active Layer)

> Алгоритм подбора опционов для ближнего слоя (Active Layer).

---

## Роль Active Layer

Ближний слой предназначен для защиты **первой фазы сильного падения**, где необходимо максимально быстро нарастить защиту при минимальных затратах. 

Поэтому лучший опцион — это не самый дешёвый и не самый "гаммовый", а тот, который:

- максимально эффективно усиливает защиту внутри требуемого диапазона;
- приносит наибольшую прибыль именно в этом диапазоне;
- имеет приемлемую стоимость содержания;
- обладает достаточной ликвидностью.

---

## Шаг 0: Определение страховочного диапазона

> **Ближний слой должен максимально эффективно работать при падении от текущей спотовой цены SOL от -3% до -10%.**

Каждый опцион мы должны оценивать **относительно этого диапазона** (Protection Range).

**Диапазон страхования:**
- Нижняя граница: S = Spot × 0.97 (падение на 3%)
- Верхняя граница: S = Spot × 0.90 (падение на 10%)

**Почему 3–10%?**
- Ниже 3% — это краткосрочный шум, хедж слишком дорогой
- Выше 10% — это уже зона, где включается Adaptation Layer (средний слой)

---

## Шаг 1: Расчёт бюджета Active Layer

См. [[200-budget-allocation]]

```
PositionSize = SOL_qty × Avg_Buy_Price
TotalBudget  = PositionSize × GlobalBudgetPct
ActiveBudget = TotalBudget × ActiveLayerPct (20%)
```

**Особенность:** Active Layer требует высокой частоты обновления. Каждый ролл — новая покупка.

## Шаг 2: Фильтрация кандидатов

Отбираем PUT-опционы по критериям из [[201-selection-criteria#active]]:

**Базовый диапазон:**
- Delta: 0.38–0.50
- DTE: 8–15 дней

## Шаг 3: Построение кривой Delta(S)

Для каждого выбранного опциона строим кривую **Delta(S)** по формуле Блэка-Шоулса в диапазоне цен базового актива **от Spot до K_85**, где K_85 — страйк, при котором delta put-опциона = 0.85.

```
Delta_put(S) = N(d1) - 1
где d1 = [ln(S/K) + (r + σ²/2)×T] / (σ×√T)
      S — цена базового актива
      K — страйк
      σ — волатильность (IV)
      T — время до экспирации
      r — безрисковая ставка
      N — функция нормального распределения
```

**Диапазон цен:** от текущей спотовой цены Spot до страйка K_85 (delta_put = 0.85), с шагом $1.

**PnL(S) — прибыль/убыток от опциона при цене S:**

```
PnL(S) = MarkPrice_BS(S, K, T, IV) - Premium
```

Где MarkPrice рассчитывается по Блэку-Шоулсу для пут-опциона:
```
MarkPrice(S) = K × e^(-rT) × N(-d2) - S × N(-d1)
где d1 = [ln(S/K) + (r + σ²/2)×T] / (σ×√T)
      d2 = d1 - σ×√T
```

**Intrinsic Value (внутренняя стоимость):**
```
Intrinsic(S) = max(K - S, 0)
```

**Time Value (временная стоимость):**
```
TimeValue(S) = MarkPrice(S) - Intrinsic(S)
```

**Gamma (Γ):**

На каждом ценовом уровне вычисляем Gamma — производную дельты по цене, скорость изменения защитной дельты:

```
Gamma(S) = N'(d1) / (S × σ × √T)
где N'(x) = (1/√(2π)) × e^(-x²/2) — функция плотности нормального распределения
      S — цена базового актива
      σ — волатильность (IV)
      T — время до экспирации
```

**Определение рабочего окна опциона:**

Рабочее окно определяется по форме кривой Gamma конкретного опциона:

1. **Находим максимум Gamma** по кривой — это центр рабочего окна (S_max)
2. **Вычисляем Gamma_80% = 0.8 × Gamma_max**
3. **Находим две точки** на кривой Gamma, где она пересекает уровень Gamma_80%:
   - **S_left** — левая граница (нижняя цена)
   - **S_right** — правая граница (верхняя цена)

**Рабочее окно опциона: [S_left, S_right]** — диапазон цен, где Gamma ≥ 80% от максимума. Это зона максимальной чувствительности опциона.

**Полная таблица кривой (Delta, Gamma, PnL, Intrinsic, Time Value):**

Для каждого уровня цены S в диапазоне [K_85, Spot] вычисляем:
- **Delta_put(S)** — дельта опциона (отрицательная для пута)
- **Gamma(S)** — гамма (скорость изменения дельты)
- **PnL(S)** = `MarkPrice(S) - Premium` — прибыль от опциона при цене S (относительно текущей стоимости покупки)
- **Intrinsic Value** = `max(K - S, 0)` — внутренняя стоимость
- **Time Value** = `MarkPrice(S) - Intrinsic Value` — временная стоимость

**Пример расчёта (Spot = 74, K = 72, DTE = 10, IV = 65%, Premium = $2.63):**

| S | Delta | Gamma | PnL | Intrinsic | Time Value |
|---|-------|-------|-----|-----------|------------|
| 74 | -0.35 | 0.045 | +0.00 | 0.00 | — |
| 73 | -0.42 | 0.052 | +0.48 | 1.00 | 0.52 |
| 72 | -0.50 | 0.055 | +1.15 | 2.00 | 0.65 |
| 71 | -0.58 | 0.051 | +1.89 | 3.00 | 1.11 |
| 70 | -0.65 | 0.044 | +2.67 | 4.00 | 1.33 |

- **Gamma_max = 0.055 при S = 72**
- **Gamma_80% = 0.8 × 0.055 = 0.044**
- **S_left ≈ 70** (где Gamma пересекает 0.044 снизу)
- **S_right ≈ 75** (где Gamma пересекает 0.044 сверху)
- **Рабочее окно: [70, 75]**

**Важно:**
- **PnL(S)** положительный при S < Spot (пут растёт при падении)
- **Intrinsic Value** = 0, когда S ≥ K (OTM), растёт при S < K
- **Time Value** максимален при S ≈ K (ATM) и уменьшается по мере ухода OTM
- Рабочее окно определяет **геометрию** (где Gamma высока), а PnL показывает **экономическую отдачу** в каждой точке

## Шаг 4: Метрики покрытия и интенсивности защиты

Coverage и Precision — это **геометрия** перекрытия. Метрика **GammaProtection** — это **интенсивность** защиты внутри перекрытия. CPD и Γ/$ — это **экономика** опциона.

### 4.1 Coverage (охват страховочного диапазона)

**Coverage** — насколько покрытие рабочего окна опциона совпадает со страховочным диапазоном (3–10%).

**Формула:**
```
Coverage = |Рабочее окно опциона ∩ Диапазон страхование (3–10%)| / |Диапазон страхование (3–10%)|
```

**Результат:** значение от 0 до 1 (или 0% до 100%).
- **Coverage = 100%** — рабочее окно опциона полностью покрывает страховочный диапазон
- **Coverage < 100%** — часть страховочного диапазона не защищена
- **Coverage = 0%** — опцион не работает в нужной зоне

**Смысл Coverage:** «Сколько % страховочного диапазона мы закрыли?»

---

### 4.2 Precision (точность покрытия)

**Precision** — какая часть рабочего окна опциона фактически используется для страховочного диапазона.

**Формула:**
```
Precision = |Рабочее окно опциона ∩ Диапазон страхование (3–10%)| / |Рабочее окно опциона|
```

**Результат:** значение от 0 до 1 (или 0% до 100%).
- **Precision = 100%** — всё рабочее окно опциона попадает в страховочный диапазон (опцион "в точку")
- **Precision < 100%** — часть рабочего окна опциона "растратилась" вне страховочного диапазона
- **Precision = 0%** — рабочее окно опциона полностью вне страховочного диапазона

**Смысл Precision:** «Сколько % нашего опциона реально работает на защиту?»

---

### 4.3 GammaProtection (Суммарная гамма защиты)

Coverage и Precision описывают **где** работает опцион. GammaProtection — это **насколько сильно** он работает.

**Формула:**
```
GammaProtection = Σ Gamma(S) × ΔS
где S ∈ [Рабочее окно опциона ∩ Диапазон страхование (3–10%)]
      ΔS = шаг $1 (или $0.5 для высокой точности)
```

**Результат:** значение от 0 до ∞ (сумма всех гамма-значений в зоне пересечения).
- **GammaProtection больше** → интенсивнее растёт дельта при падении → эффективнее хедж
- **GammaProtection меньше** → опцион слабее реагирует на движения в зоне защиты

**Смысл GammaProtection:** «Какая суммарная защитная сила опциона в зоне страховки?»

**Пример расчёта:**

| Цена S | Gamma | В страховке? |
|--------|-------|-------------|
| 66     | 0.041 | ✅ Да       |
| 67     | 0.043 | ✅ Да       |
| 68     | 0.045 | ✅ Да       |
| 69     | 0.046 | ✅ Да       |
| 70     | 0.047 | ✅ Да       |
| 71     | 0.047 | ✅ Да       |

**GammaProtection = (0.041 + 0.043 + 0.045 + 0.046 + 0.047 + 0.047) × $1 = 0.269**

---

### 4.4 CPD и Γ/$ (экономика опциона)

Ближний слой — **расходный материал**. Каждый день опцион теряет стоимость из-за theta. Важно знать, **сколько защиты мы получаем на каждый доллар premiums**.

**Cost per Day (CPD):**
```
CPD = Premium / DTE
```

**Смысл:** сколько денег мы теряем каждый день, удерживая опцион.

**GammaProtection per Dollar (Γ/$):**
```
Γ/$ = GammaProtection / Premium
```

**Смысл:** сколько суммарной защитной силы (гаммы) мы получаем на каждый вложенный доллар.

**Чем выше Γ/$ — тем выгоднее опцион.**

**Пример:**
| Опцион | Premium | DTE | CPD | GammaProtection | Γ/$ |
|--------|---------|-----|-----|----------------|-----|
| 72-P | $2.73 | 10 | $0.27 | 0.2698 | **0.099** |
| 74-P | $3.69 | 10 | $0.37 | 0.2088 | 0.057 |

72-P на **75% эффективнее** по Γ/$.

---

## Шаг 5: WeightedPnL (взвешенная полезность кривой PnL)

Вместо сравнения PnL в одной точке, строим **всю кривую PnL** внутри пересечения рабочего окна и страховочного диапазона, и суммируем приросты PnL с весами.

**Почему это важно для Active Layer:** первые проценты падения рынка важнее последних. Именно в зоне 3–5% опцион начинает давать отдачу, когда средний слой ещё не включён. Чем раньше и быстрее растёт PnL — тем эффективнее опцион.

**Формула:**
```
WeightedPnL = Σ ΔPnL_i × weight_i

где:
  - Диапазон точек S: от Spot до S_left (пересечение рабочего окна и страховочного диапазона), шаг $1
  - ΔPnL_i = PnL(S_i) - PnL(S_{i-1}) — прирост PnL относительно предыдущей точки
  - weight_i — вес точки (чем выше точка, тем больше вес)
```

**Расчёт весов:**
- Первый шаг от Spot вниз: weight = 1.00
- Шаг весов: 0.05
- weight_i = 1.00 + (N - i) × 0.05, где N — общее число шагов, i — текущий индекс от верха
- Пример: Spot → Spot-1 → Spot-2 → ...
  - Spot - 1$: weight = 1.05
  - Spot - 2$: weight = 1.10
  - Spot - 3$: weight = 1.15
  - ...

**Пример расчёта:**

| Шаг | Цена S | PnL | ΔPnL | Вес | ΔPnL × Вес |
|-----|--------|-----|------|-----|------------|
| Spot | 74.00 | 0.00 | — | 1.00 | — |
| 1 | 73.00 | +0.48 | +0.48 | 1.05 | +0.504 |
| 2 | 72.00 | +1.15 | +0.67 | 1.10 | +0.737 |
| 3 | 71.00 | +1.89 | +0.74 | 1.15 | +0.851 |
| 4 | 70.00 | +2.67 | +0.78 | 1.20 | +0.936 |
| 5 | 69.00 | +3.50 | +0.83 | 1.25 | +1.038 |

**WeightedPnL = 0.504 + 0.737 + 0.851 + 0.936 + 1.038 = 4.066**

**Смысл WeightedPnL:** «Какая общая полезная отдача опциона с учётом, что ранние потери при падении важнее поздних»

**Примечание:** PnL считается как `MarkPrice(S) - Premium`. Положительный PnL при падении означает прибыль от роста стоимости пута.

## Шаг 6: Сравнение эффективности (Efficiency)

Дополнительная метрика для финальной сортировки.

```
PnL_at_target = BS_Price(S_target, K, T, IV) - Premium
Efficiency = PnL_at_target / Premium
```

Где:
- **S_target** — целевая цена в страховочном диапазоне (например, S = Spot × 0.90)
- **Premium** — текущая цена опциона
- **BS_Price** — Black-Scholes цена опциона при целевой цене

**Сравниваем средний PnL** из части или полного страховочного диапазона, которые находятся **внутри** рабочего окна опциона.

## Шаг 7: Выбор опциона и расчёт количества

### Выбор страйка

**Полный алгоритм выбора (6 уровней приоритета):**

| Приоритет | Метрика | Цель | Что оценивает |
|-----------|---------|------|---------------|
| **1** | **Coverage** | 100% | Геометрия: полное покрытие страховочного диапазона |
| **2** | **GammaProtection** | Максимум | Интенсивность: суммарная защитная сила в зоне перекрытия |
| **3** | **Γ/$** | Максимум | Экономика: защита на каждый вложенный доллар |
| **4** | **Precision** | Максимум | Геометрия: минимальный избыток |
| **5** | **WeightedPnL** | Максимум | Эффективность: полезная отдача с учётом важности ранних потерь |
| **6** | **CPD** | Минимум | Экономика: минимальные ежедневные убытки |

```
K_optimal = argmax по всем метрикам с приоритетами 1→6
```

**Ключевые отличия метрик:**
- Coverage и Precision — это **геометрия** (сколько процентов диапазона покрыто)
- GammaProtection — это **интенсивность** (насколько сильно опцион защищает в зоне покрытия)
- **Γ/$** — это **экономическая эффективность** (сколько защиты на каждый доллар premiums)
- CPD — это **стоимость владения** (сколько теряем в день)
- **WeightedPnL** — это **временная ценность** (отдача с учётом важности первых процентов падения)

**Приоритеты:**
1. Coverage 100% → GammaProtection → **Γ/$** → Precision → WeightedPnL → CPD
2. Опцион с Precision 80% и GammaProtection 0.35 лучше, чем Precision 100% и GammaProtection 0.20
3. Опцион с GammaProtection 0.25 и Γ/$ 0.09 лучше, чем GammaProtection 0.30 и Γ/$ 0.04 (дешевле и эффективнее)

**Характеристики:**
- Опцион должен работать именно в зоне средних коррекций
- Не обязательно ATM — первый OTM часто эффективнее (шире покрытие)

### Расчёт количества контрактов

**Бюджет:**
```
TotalCost = n × Premium ≤ ActiveBudget
```

**Delta-хедж** при падении в страховочном диапазоне (S = Spot × 0.90):
```
Delta_hedge = n × |Delta(S=0.90×Spot)|
DeltaRatio = Delta_hedge / SOL_qty
```

**Целевая дельта:** >= 20–30% при падении на 10% (Active Layer — усилитель, а не полная защита).

**Оптимизация:**
Ищем n, которые:
1. Укладываются в ActiveBudget
2. DeltaRatio >= 20–30% при падении на 10%
3. Максимально покрывают страховочный диапазон (3–10%)

## 🔗 Связь с [[104-active-closing-rules]]: условия закрытия

Все условия закрытия из [[104-active-closing-rules]] переосмысливаются через конкретные метрики:

| Критерий из [[104-active-closing-rules]] | Метрика | Что происходит |
|-----------------|---------|----------------|
| **Delta > 0.80** | Delta | Цена вышла из рабочей зоны сверху — гамма мертва |
| **Delta > 0.90** | Delta | Опцион глубоко ITM — рабочая зона исчерпана |
| **Delta < 0.15** | Delta | Цена ушла глубоко OTM — гамма мертва |
| **DTE < 4** | DTE | Осталось мало времени — theta-распад убивает стоимость |
| **Рынок упал на 10–15%** | Distance | Цена ушла за границу рабочей зоны → ролл к ATM |
| **Смена тренда** | Delta, Gamma | Цена вышла из зоны работы → Close (не ролл) |

**Главный принцип:** закрытие — это не фиксация прибыли/убытка, это реакция на выход из рабочей зоны.
