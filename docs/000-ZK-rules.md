# 📏 Правила Zettelkasten для HedgeModel

> Правила организации заметок. Обязательны к соблюдению.

---

## Принцип

Одна заметка = одна идея. Без дублирования. Связи через `[[wiki-links]]`.

## Структура

- Плоская структура, без подпапок
- Префиксы по контексту: `000-` (навигация), `100-` (стратегия), `200-` (правила), `300-` (скрипты), `400-` (операции), `500-` (отладка), `900-` (исследования)

## Правила заметок

1. **Одна идея** — не смешивай стратегию и формулы
2. **YAML-шапка** — тип, статус, связи, даты
3. **Wiki-ссылки** — связываешь с другими заметками
4. **Нет дублирования** — ссылка вместо копирования
5. **Ссылка на источник** — откуда взялась информация
6. **Статус** — `draft` / `active` / `deprecated`
7. **Дата** — `created` и `updated`

## YAML-шапка

```yaml
---
type: strategy | rule | formula | script | research | ops | MOC | rules
status: draft | active | deprecated
related: [[note-id]], [[note-id]]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

## Типы заметок

| Тип | Описание | Пример |
|-----|----------|--------|
| `strategy` | Философия, архитектура, принципы | 100-strategy |
| `rule` | Правила, критерии, ограничения | 201-selection-criteria |
| `formula` | Формулы, расчёты | 202-formulas |
| `script` | Описание скриптов | 301-optionboard-script |
| `research` | TODO, гипотезы, вопросы | 900-research-backlog |
| `ops` | Операции, деплой, мониторинг | 400-deployment |
| `MOC` | Карта всех заметок | 000-MOC |
| `rules` | Правила организации | 000-ZK-rules |

## Как добавлять заметки

1. Придумай название для одной идеи
2. Выбери префикс по контексту
3. Напиши заметку с YAML-шапкой
4. Связи через `[[wiki-links]]`
5. Добавь ссылку в MOC

## Как читать заметки

1. Начни с [[000-MOC]]
2. Перейди к нужной заметке по ссылке
3. В заметке переходи по `related` для контекста
4. Если ссылка битая — создай заметку
