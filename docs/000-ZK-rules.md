# 000-ZK-rules — Правила Zettelkasten

> Общие правила для всех заметок проекта.

## 📁 Структура каталогов

| Префикс | Тип | Назначение |
|---------|-----|------|
| `000-` | правила | Правила проекта (уникален) |
| `100-` | концепции | Стратегия, архитектура, идеи |
| `200-` | критерии | Правила, параметры, ограничения |
| `300-` | процессы | Пайплайны, инструкции, workflow |
| `400-` | данные | Источники, API, форматы |
| `500-` | эксперименты | Тесты, результаты, выводы |
| `900-` | задачи | Открытые задачи |

## 📝 Формат заметки

```markdown
# NN-название-заметки

tags: [тег1, тег2]
created: YYYY-MM-DD
references:
  - [[NN-название]]
  - [[NN-название#заголовок]]

Содержимое заметки...
```

## 🔗 Правила именования

- Файлы на **английском**
- Префикс `NDD-` (001, 002, 010, 020...)
- Дефис между номером и названием: `000-ZK-rules`
- Без пробелов в названии

## 🔗 Правила ссылок

- В тексте: `[[NN-название-файла]]`
- С якорем: `[[NN-название#заголовок-подсекции]]`
- Внешние ссылки: полный URL `https://...`
- При создании заметки — минимум 1 ссылка на существующую

## 📏 Принципы

- Одна заметка — одна идея
- Уникальный ID для каждой заметки
- Всегда создавать связи, а не просто писать
- Не дублировать информацию — использовать ссылки
- Старые заметки не удалять, а обновлять
