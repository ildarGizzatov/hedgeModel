# T009 — Алгоритм выбора опционов ближнего слоя (Active Layer)

> Определить алгоритм подбора опционов для Active Layer (Profit Engine).

## 📌 Описание

Active Layer — ближний слой хеджа: OTM → ATM PUT-опционы с коротким DTE (5–10 дней).

**На практике ближний слой — это не страховка в классическом смысле, а динамический усилитель дельты в зоне текущей цены.**

Ближний слой — постоянно поддерживаемый инструмент защиты, а не спекулятивная сделка.

**Ключевой принцип:** для активного слоя важнее всего время и оставшаяся эффективность. Закрываем, когда цена времени превышает оставшуюся полезность опциона.

## 🔑 Вопросы для решения

### 1. Критерии выбора опционов

Active Layer подбирает опционы из диапазона:
- Delta: 0.38–0.55
- DTE: 5–10 дней
- Distance от спота: 0–5%

**Как отбирать?**
- Приоритет по Gamma (максимальная чувствительность к движению)?
- Приоритет по Efficiency (PnL/$)?
- Комбинация?

### 2. Подбор при движении рынка

- Рынок упал на 10–15% — роллируем на опционы ближе к ATM. Как выбрать конкретные страйки?
- Если рынок резко вырос — быстро выходим (Delta падает). Нужно ли обновлять позиции на новые опционы?

### 3. Постоянно поддерживаемый инструмент

Ближний слой должен всегда оставаться рядом с рынком. Это значит:
- При движении цены нужно роллировать или покупать новые опционы
- Как часто пересматривать состав слоя?
- Какой минимальный размер позиции по слою?

### 4. Взаимодействие с T008 (условия закрытия)

Критерии закрытия уже определены:
- Delta > 0.80 → закрыть
- Delta > 0.90 → закрыть (особенно критично)
- Delta < 0.15 → закрыть
- DTE < 4 → закрыть
- Рынок упал 10–15% → ролл ближе к ATM
- Смена тренда → Close

**Вопрос:** как алгоритм выбора должен учитывать условия закрытия? Например, не покупать опционы, которые быстро достигнут Delta > 0.80.

### 5. Бюджет и количество контрактов

- ActiveLayer получает 20% от глобального бюджета
- Как распределять между опционами?
- Сколько опционов в слое одновременно?

## 🎯 Основа: Гамма-окно

> **Ближний слой существует только пока цена находится в зоне, где дельта может быстро изменяться (за счёт гаммы).**

### Гамма-окно — главный критерий

**Гамма-окно** — это диапазон цен, в котором опцион ещё способен ускорять защиту за счёт роста гаммы.

```
Движение цены → усиливает дельту → усиливает хедж → усиливает эффект движения
```

**Важно только одно: жива ли гамма-зона.**

Не важно:
- ❌ Есть ли прибыль
- ❌ Есть ли убыток
- ❌ Сколько осталось DTE

**Критерий один:** цена в гамма-окне → держим (или роллим). Цена вышла за пределы гамма-окна → закрываем.

## 📐 Практика: Гамма-окно как инструмент отбора

> **Для каждого страйка выстроить «гамма-окно»** и оценивать: попадает ли страйк в окно, какой DTE даёт нужную дельту и гамму, и какова efficiency.

### Шаг 1: Вычислить гамма-окно для каждого страйка

Для каждого доступного страйка определить **диапазон цен**, где gamma > порог (например, gamma > 50% от максимума). Это и есть гамма-окно страйка.

### Шаг 2: Добавить разные DTE

Разные DTE дают разную форму гамма-окна:
- **Короткий DTE (5-7 дней):** узкое, острое окно — гамма высокая, но быстро сужается
- **Средний DTE (8-10 дней):** шире окно — гамма чуть ниже, но дольше держится

### Шаг 3: Оценить эффективность нахождения в окне

Для каждого страйка оценить:
- **Находится ли текущая цена в гамма-окне?** (да/нет/на границе)
- **Какова текущая дельта и гамма** в этой зоне
- **Какой потенциал:** насколько быстро дельта растёт при движении вниз
- **Насколько широко окно:** сколько «воздуха» между текущей ценой и границей

### Шаг 4: Добавить Efficiency

Внутри подходящих страйков выбрать по efficiency:
```
Efficiency = PnL_hedge_at_target / TotalHedgeCost
```

**Результат:** матрица, где каждый страйк имеет:
1. Гамма-окно (диапазон цен)
2. DTE (форма окна)
3. Delta/Gamma в текущей зоне
4. Потенциал (ускорение дельты)
5. Efficiency (экономическая эффективность)

## 🔗 Связь с T008: условия закрытия через гамма-окно

Все условия закрытия из [[104-active-closing-rules]] переосмысливаются через гамма-окно:

| Критерий из T008 | Через призму гамма-окна |
|-----------------|------------------------|
| **Delta > 0.80** | Цена вышла из гамма-зоны сверху — гамма мертва, дельта стабильна |
| **Delta > 0.90** | Опцион глубоко ITM — гамма-окно исчерпано полностью |
| **Delta < 0.15** | Цена ушла глубоко OTM — гамма мертва, дельта близка к 0 |
| **DTE < 4** | Осталось мало времени — гамма-окно сужается, theta-распад убивает стоимость |
| **Рынок упал на 10–15%** | Цена ушла за нижнюю границу гамма-окна → ролл к ATM (возврат в гамма-зону) |
| **Смена тренда** | Цена вышла из зоны, где гамма работала → Close (не ролл) |

**Главный принцип:** закрытие — это не фиксация прибыли/убытка, это реакция на выход из гамма-зоны.

### Уровень 1: Автоматический анализ (система оценивает)
- Текущая Delta ближайших PUT
- DTE доступных опционов
- IV в динамике (растёт/падает?)
- Расстояние от спота
- Gamma доступных опционов
- Time decay (DTE < 4?)

### Уровень 2: Антипаттерны (когда НЕ стоит покупать)

- ❌ **В середине флэта** — нет движения = theta съедает премию
- ❌ **После резкого падения** — уже поздно, Delta уже высока, Gamma упала
- ❌ **При низкой волатильности (низкий IV)** — нет разгона премии, опционы дёшевы, но и смысла нет

## 💰 Бюджет и частота обновления

> Ближний слой обычно получает 20% от общего бюджета хеджа, но с **высокой частотой обновления**.

Это означает:
- Меньший бюджет, но чаще пересматриваем состав
- Каждый ролл — новая покупка, theta-распад требует быстрой ротации
- Нельзя «купил и забыл» — слой должен постоянно корректироваться под текущую цену

- [ ] Рынок перегрет на локале? (да/нет)
- [ ] IV растёт после движения? (да/нет)
- [ ] Видишь разворот или просто коррекция? (разворот/коррекция/не уверен)
- [ ] Готов тратить X% бюджета сейчас? (да/нет/другая сумма)
