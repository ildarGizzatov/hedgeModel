# 📋 T003 — Условия экономической эффективности роллирования опционов

> Принимать решение о ролле только если чистая экономия положительна.

## 📌 Описание

При роллировании опционов среднего слоя (Adaptation Layer) возникают многосторонние затраты:

1. **Закрытие текущей позиции** — убыток от закрытия по текущей цене (если опцион не безубыточен)
2. **Открытие новой позиции** — премия нового опциона
3. **Комиссия биржи** — двусторонняя комиссия (закрытие + открытие)
4. **Спред bid/ask** — потери от рыночного исполнения
5. **Стоимость содержания** — ежедневный тета-распад нового опциона
6. **Налоговые последствия** — фиксация убытка/прибыли

**Принцип:** Роллируем только если `ЧистаяЭкономия > 0`.

## 📊 Формула ролл-решения

```
ЧистаяЭкономия = ЭкономияОтРолла − ЗатратыОтРолла

ЗатратыОтРолла = ЗакрытиеПозиции + КомиссияЗакрытие + КомиссияОткрытие + СпредЗатраты

ЭкономияОтРолла = (ТетаДнейБезРолла) − (ТетаНовогоОпциона × ДнейНовогоОпциона)
                 + (РазницаВСтраике × ДельтаПозиции)
                 + (СтоимостьСодержанияБезРолла − СтоимостьСодержанияСРоллом)
```

### Затраты от закрытия
```
ЗатратыЗакрытия = max(0, (Strike − CurrentPrice) × Multiplier × Contracts)
```

### Затраты от открытия
```
ЗатратыОткрытия = PremiumNew × Multiplier × Contracts
```

### Комиссии
```
Комиссия = (BidPrice + AskPrice) / 2 × Multiplier × Contracts × CommissionRate
```

### Тета-экономика
```
ТетаДнейБезРолла = ThetaOld × DTE_old
ТетаНовогоОпциона = ThetaNew / DTE_new
```

## 🔍 IV-фильтр (предварительный)

Ролл допускается **только если**: `IV_нового_опциона ≤ IV_при_входе_старого_опциона`

- `IV_при_входе` берётся из `data/options_registry.csv` (поле `iv_entry`)
- Если IV нового опциона выше — ролл **не проводим** (не платим за рост стоимости волатильности)
- Если IV_entry отсутствует в реестре — ролл **не проводим** до его внесения
- Фильтр применяется **до** расчёта ЧистаяЭкономия

## 🎯 Критерий ролла

```
РоллВЫГОДЕН, если:
  ЧистаяЭкономия > ЗатратыНаТранзакцию × (1 + МаржаЗаЩита)
```

Где `МаржаЗаЩита` — buffer на непредвиденные движения рынка при исполнении ролла.

## 📐 Сценарии

### Сценарий 1: Ролл по горизонтали (DTE)
- Закрываем опцион с малым DTE
- Покупаем опцион с большим DTE
- **Выгодно:** если тета-экономия от длинного DTE перекрывает затраты на разницу премий

### Сценарий 2: Ролл по вертикали (Strike)
- Закрываем текущий страйк
- Покупаем другой страйк
- **Выгодно:** если новый страйк даёт лучшую защиту за ту же или меньшую цену

### Сценарий 3: Ролл по горизонтали + вертикали
- Комбинированный ролл
- **Выгодно:** только если суммарная экономия значительно превышает затраты

## 📊 Выходные данные

- Таблица сравнения: `ТекущаяПозиция → НовыйОпцион`
- Разбивка всех затрат
- Чистая экономия в $ и % от бюджета слоя
- Рекомендация: Роллировать / Закрыть / Держать
